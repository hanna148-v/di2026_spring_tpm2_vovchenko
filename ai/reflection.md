## Часть 4. Коммуникация и рефлексия (дополнительно)
### 4.1. Вопросы стейкхолдеру
1. **"Какой результат вы считаете успехом для этой AI‑функции?"**  
Это проясняет, что для заказчика важнее: снижение нагрузки на операторов, рост CSAT, уменьшение негативных отзывов, экономия на поддержке или скорость реакции.
2. **"Какие ограничения по данным и интеграциям у нас есть на старте?"**  
Этот вопрос вскрывает: какие API доступны, какие данные можно отдавать AI, есть ли ограничения по персональным данным, насколько готов backend. Это напрямую влияет на MVP и сроки.

2. **"Что для нас важнее в первые 3 месяца: чтобы ИИ идеально отвечал на текстовые вопросы по правилам сервиса (FAQ) или чтобы он имел "кривой", но рабочий доступ к статусам рейсов и кнопке возврата?"**  
Это ответ на риск Т1 о недостатке данных. Ответ поможет распределить ресурсы: нанимать редакторов для обучения модели текстам или бросить все силы бэкенд-разработчиков на создание API-шлюза для транзакций.

3. **"Все ли перевозчики-партнеры согласны на автоматический возврат средств через ИИ, или нам нужно внедрить "белый список" только для тех, кто работает по прямым договорам с Atlasbus?"**  
У аtlasbus много партнеров. Если ИИ вернет деньги за рейс частного перевозчика, который не признает автоматические отмены, это приведет к кассовому разрыву.
